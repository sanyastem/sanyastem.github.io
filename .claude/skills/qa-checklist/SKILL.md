Создай чеклист тестирования для: `$ARGUMENTS`

Если аргумент — путь к файлу, проанализируй код и создай чеклист на основе реальной логики.
Если аргумент — описание, используй его.
Если аргумент не передан, спроси у пользователя что тестируем.

## Категории чеклиста

### Функциональность
- Основной сценарий (happy path)
- Альтернативные сценарии
- Обработка ошибок

### Валидация данных
- Обязательные поля
- Форматы (email, телефон, дата)
- Граничные значения (min, max, пустое)
- Спецсимволы, Unicode, эмодзи

### UI/UX
- Отображение на разных разрешениях
- Состояния элементов (disabled, loading, error, empty)
- Порядок табуляции
- Сообщения об ошибках — понятны пользователю?

### Безопасность
- Доступ без авторизации
- Доступ к чужим данным
- Инъекции (SQL, XSS)
- CSRF

### Производительность
- Загрузка при большом объёме данных
- Множественные быстрые клики
- Таймауты

### Совместимость
- Chrome, Firefox, Safari, Edge
- Мобильные устройства
- Разные ОС

## Формат вывода

Чеклист с чекбоксами:

- [ ] Категория: Проверка
