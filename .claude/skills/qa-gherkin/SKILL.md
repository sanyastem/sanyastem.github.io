Сгенерируй Gherkin-сценарии для: `$ARGUMENTS`

Если аргумент — путь к файлу, проанализируй логику и создай сценарии.
Если аргумент — user story или описание, используй его.
Если аргумент не передан, спроси описание фичи.

## Правила написания сценариев

### Feature
Одна фича — один файл. Краткое описание на первой строке.

### Scenario
Каждый сценарий — одна конкретная ситуация. Название описывает ЧТО проверяем.

### Given (Допустим)
Начальное состояние системы. Что уже настроено/существует.

### When (Когда)
Действие пользователя. Одно действие на строку.

### Then (Тогда)
Ожидаемый результат. Что должно произойти.

### Scenario Outline (Структура сценария)
Для параметризованных тестов с таблицей Examples.

## Что включить

1. Happy path — основной сценарий
2. Альтернативные пути — другие варианты использования
3. Негативные сценарии — ошибки, невалидные данные
4. Граничные случаи — лимиты, пустые значения

## Формат вывода

```gherkin
# language: ru

Функционал: Название фичи
  Как [роль]
  Я хочу [действие]
  Чтобы [цель]

  Сценарий: Описание
    Допустим [предусловие]
    Когда [действие]
    Тогда [результат]
```

Язык сценариев: русский (по умолчанию) или английский (если пользователь попросит).
