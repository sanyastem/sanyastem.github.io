Оформи баг-репорт из описания: `$ARGUMENTS`

Если передан путь к файлу с кодом, попробуй найти причину бага в коде и указать файл и строку.
Если аргумент не передан, спроси у пользователя описание бага.

## Формат баг-репорта

### Заголовок
Краткий, конкретный, начинается с действия. Пример: "500 ошибка при отправке формы с пустым email"

### Severity
- **Blocker** — система недоступна, нет workaround
- **Critical** — основная функция не работает, есть workaround
- **Major** — функция работает неправильно, но не блокирует
- **Minor** — косметика, опечатки, мелкие неудобства

### Шаги воспроизведения
Пронумерованные, конкретные. Каждый шаг — одно действие.

### Ожидаемый результат
Что должно было произойти.

### Фактический результат
Что произошло на самом деле. Включи: текст ошибки, HTTP-код, скриншот (если описан).

### Окружение
Браузер, ОС, окружение (dev/staging/prod), разрешение экрана.

### Дополнительно
- Частота воспроизведения (всегда / иногда / один раз)
- Workaround (если есть)
- Связанные тест-кейсы
- Файл и строка в коде (если найдена причина)
