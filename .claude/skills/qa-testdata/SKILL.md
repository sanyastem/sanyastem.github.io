Сгенерируй тестовые данные для: `$ARGUMENTS`

Если аргумент — путь к файлу (модель, контроллер, схема), проанализируй валидации и ограничения в коде.
Если аргумент — описание полей, используй его.
Если аргумент не передан, спроси что нужно.

## Категории данных

### Валидные данные
Корректные значения, которые система должна принять.

### Невалидные данные
Значения, которые система должна отклонить: пустые, слишком длинные, неправильный формат.

### Граничные значения
Минимальная и максимальная длина, 0, отрицательные числа, значения на границе.

### Специальные символы
Unicode, эмодзи, HTML-теги, SQL-запросы, скрипты (XSS).

### Реалистичные данные
Правдоподобные имена, адреса, email для разных стран и локалей.

## Форматы вывода

В зависимости от запроса выдай данные в нужном формате:
- **Таблица** — для ручного тестирования
- **JSON** — для API-тестов
- **CSV** — для импорта
- **SQL INSERT** — для наполнения БД

Для каждого значения укажи: для какого теста оно предназначено (позитивный / негативный / граничный).
