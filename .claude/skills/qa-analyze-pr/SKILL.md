Проанализируй изменения и оцени риски: `$ARGUMENTS`

Если аргумент не передан, выполни `git log --oneline -10` и `git diff HEAD~1` чтобы увидеть последние изменения.

## Порядок анализа

### 1. Что изменилось
Перечисли изменённые файлы и кратко опиши суть каждого изменения.

### 2. Затронутые фичи
Определи, какие пользовательские сценарии могут быть затронуты этими изменениями.

### 3. Risk Assessment

Для каждого изменения оцени:
- **Риск** (high / medium / low)
- **Почему** — что может сломаться
- **Где проверить** — конкретные страницы/эндпоинты

### 4. Рекомендации по тестированию

- Какие тест-кейсы прогнать обязательно
- Какие области проверить на регрессию
- На что обратить особое внимание

### 5. Потенциальные проблемы

Проверь изменённый код на:
- Отсутствие валидации входных данных
- Отсутствие обработки ошибок
- Хардкод значений
- Проблемы с безопасностью

## Формат вывода

Структурированный отчёт с таблицей рисков и списком рекомендаций.
