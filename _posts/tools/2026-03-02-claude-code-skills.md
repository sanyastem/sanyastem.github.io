## Что такое Skills

Skills (навыки) — это Markdown-файлы с инструкцией для агента. Пишешь один раз, дальше используешь в любой сессии. Вместо того чтобы каждый раз объяснять «сделай код-ревью по нашим стандартам» — один раз пишешь skill и набираешь `/review`.

## Skills и slash-команды — теперь одно

В 2026 Anthropic объединила две сущности, которые раньше были раздельными:

- **Slash-команды** (старый формат `.claude/commands/*.md`) — вызываешь вручную через `/имя`
- **Agent Skills** (новый формат `.claude/skills/<name>/SKILL.md`) — Claude может вызвать **сам**, когда задача подходит под `description`

Теперь это **одна система**: файлы в обеих папках создают `/команду`. Разница в том **кто вызывает**:

| | Кто запускает | Когда использовать |
|---|---|---|
| **User-invoked** | Ты вручную `/deploy` | Действия с побочным эффектом, которые ты контролируешь |
| **Agent-invoked** | Claude сам по `description` | Навыки которые модель применяет автоматически (форматирование, проверки) |

Один `SKILL.md` даёт **оба**: и авто-триггер по описанию, и ручную `/команду`. Управляешь поведением через frontmatter (`user-invocable`, `disable-model-invocation`).

## Где хранятся Skills

Рекомендуемый формат — папка с `SKILL.md`:

```
.claude/
└── skills/
    ├── review/
    │   └── SKILL.md     # /review
    ├── commit/
    │   └── SKILL.md     # /commit
    └── deploy-check/
        └── SKILL.md     # /deploy-check
```

Старый формат `.claude/commands/*.md` тоже работает (обратная совместимость), но новые skills лучше класть в `.claude/skills/` — он поддерживает frontmatter, вспомогательные файлы рядом и авто-вызов агентом.

Глобальные skills (для всех проектов):

```
~/.claude/
└── skills/
    └── standup/
        └── SKILL.md     # /standup — везде
```

## Структура файла — SKILL.md с frontmatter

```markdown
---
name: review
description: Код-ревью изменений по стандартам проекта
allowed-tools: Read, Bash
argument-hint: "[путь к файлу или пусто для всех изменений]"
effort: medium
---

Сделай код-ревью всех изменённых файлов.
Проверь: нет ли багов, соблюдены ли наши соглашения из CLAUDE.md,
нет ли явных проблем с производительностью.
Оформи как список замечаний с указанием файла и строки.
```

Сохрани как `.claude/skills/review/SKILL.md` — и теперь в сессии доступна команда `/review`.

**Ключевые поля frontmatter:**

| Поле | Что делает |
|------|-----------|
| `description` | Описание (до 1024 символов) — Claude понимает когда применять |
| `allowed-tools` | Ограничить инструменты (Read, Edit, Write, Bash...) |
| `argument-hint` | Подсказка какие аргументы принимает |
| `effort` | `low` / `medium` / `high` — глубина проработки (больше = больше токенов) |
| `context: fork` | Запустить в субагенте (изоляция) |
| `user-invocable: false` | Только Claude может вызывать, не пользователь |

## Пример: skill для коммитов

```markdown
# Commit

Создай git-коммит для всех staged изменений.

1. Запусти `git diff --staged` чтобы увидеть что меняется
2. Придумай сообщение в формате Conventional Commits:
   - feat: новая функция
   - fix: исправление бага
   - refactor: рефакторинг без изменения поведения
   - docs: только документация
3. Запусти `git commit -m "сообщение"`
4. Покажи итоговый коммит через `git log -1`
```

## Пример: skill для проверки перед деплоем

```markdown
# Deploy check

Перед деплоем проверь:

1. `npm run lint` — нет ошибок линтера
2. `npm run test` — все тесты зелёные
3. `npm run build` — сборка проходит
4. Нет незакоммиченных изменений (`git status`)

Если что-то упало — останови и сообщи что именно.
Если всё ок — напиши "Готово к деплою ✓"
```

## Аргументы в Skills

Skills могут принимать аргументы через `$ARGUMENTS`:

```markdown
# Fix issue

Посмотри на GitHub Issue #$ARGUMENTS.
Найди связанный код в репозитории и предложи исправление.
```

Вызов: `/fix-issue 142`

## Пример: skill для дейлистендапа

```markdown
# Standup

Помоги написать сообщение для дейлистендапа.

1. Запусти `git log --since="yesterday" --oneline --author="$(git config user.name)"`
2. На основе коммитов сформируй текст:
   - **Вчера:** что сделал (из коммитов)
   - **Сегодня:** что планирую (спроси у меня)
   - **Блокеры:** есть ли что-то что мешает

Формат — короткие пункты, без лишней воды.
```

## Встроенные Skills

В Claude Code есть встроенные команды и bundled skills:

| Команда | Что делает |
|---------|-----------|
| `/help` | Список всех команд |
| `/clear` | Сбросить контекст сессии |
| `/compact` | Сжать историю диалога |
| `/memory` | Управление CLAUDE.md файлами памяти |
| `/doctor` | Диагностика установки |
| `/rewind` | Откат к предыдущему checkpoint |
| `/batch` | Параллельные изменения в большой кодовой базе |
| `/loop` | Повторять задачу по расписанию (`/loop 5m /check`) |
| `/simplify` | Ревью изменений на качество и эффективность |

## Итого

- Новый формат: `.claude/skills/<name>/SKILL.md` с frontmatter
- Старый формат `.claude/commands/*.md` — тоже работает
- `$ARGUMENTS` — для передачи параметров
- `allowed-tools` в frontmatter — ограничивает что агент может делать
- `effort: high` — для сложных задач которые требуют глубокого анализа
