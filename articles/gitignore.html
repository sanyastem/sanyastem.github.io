.gitignore: что туда класть и почему

Запушить .env с паролями или node_modules на 300 МБ — классические ошибки новичков. Разбираем раз и навсегда что должно быть в .gitignore.

Что такое .gitignore

Файл .gitignore говорит Git какие файлы и папки игнорировать — не отслеживать, не добавлять в коммиты. Создаётся в корне репозитория.

Что обязательно игнорировать

Зависимости

node_modules, vendor, .venv — эти папки генерируются автоматически командой npm install, pip install и т.д. Они могут весить сотни мегабайт и не нужны в репо — у каждого разработчика они свои.

Файлы окружения

.env — здесь хранятся секреты: пароли, API-ключи, токены. Если закоммитишь — они попадут в публичный репо. Это серьёзная уязвимость.

⚠️

Если ты уже закоммитил .env — простое добавление в .gitignore не поможет. Нужно удалить файл из истории и сменить все скомпрометированные ключи.

Системные файлы

.DS_Store (macOS), Thumbs.db (Windows) — мусор от операционной системы, не нужен в репо.

Артефакты сборки

dist/, build/, .next/, __pycache__/ — генерируются при сборке, не коммитятся.

Типовой .gitignore для Node.js проекта

# Зависимости
node_modules/

# Окружение
.env
.env.local
.env.*.local

# Сборка
dist/
build/
.next/

# Логи
*.log
npm-debug.log*

# Системные
.DS_Store
Thumbs.db

# Редакторы
.vscode/settings.json
.idea/
💡

Сайт gitignore.io генерирует готовый .gitignore по технологиям. Введи «Node», «Python», «macOS» — получишь полный файл.

Глобальный .gitignore

Системный мусор (.DS_Store, .idea/) лучше вынести в глобальный gitignore — чтобы не добавлять его в каждый проект:

# Создать глобальный gitignore
touch ~/.gitignore_global

# Прописать в git конфиге
git config --global core.excludesfile ~/.gitignore_global

# Добавить системные файлы
echo ".DS_Store" >> ~/.gitignore_global
echo ".idea/" >> ~/.gitignore_global

Если файл уже в репо

Добавление в .gitignore не удаляет файл из Git — он уже отслеживается. Чтобы убрать из отслеживания, но оставить локально:

# Убрать папку из отслеживания (файлы останутся локально)
git rm -r --cached node_modules/
git rm --cached .env

# Закоммитить удаление из отслеживания
git commit -m "remove tracked files from gitignore"
