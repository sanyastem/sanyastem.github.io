GitHub Actions: автодеплой за 15 минут

GitHub Actions — встроенный CI/CD в каждом репозитории GitHub. Бесплатно до 2000 минут в месяц для публичных репо. Настраивается одним YAML-файлом.

Как это работает

Ты создаёшь файл .github/workflows/deploy.yml — и GitHub автоматически запускает его при пуше, создании PR или по расписанию. Внутри описываешь шаги: установить зависимости, запустить тесты, задеплоить.

Структура workflow-файла

# .github/workflows/deploy.yml

name: Deploy

# Когда запускать
on:
  push:
    branches: [ main ]
  pull_request:
    branches: [ main ]

# Что делать
jobs:
  build-and-deploy:
    runs-on: ubuntu-latest

    steps:
      # Скачать код репозитория
      - uses: actions/checkout@v4

      # Установить Node.js
      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'

      # Установить зависимости
      - name: Install dependencies
        run: npm ci

      # Запустить тесты
      - name: Run tests
        run: npm test

      # Собрать проект
      - name: Build
        run: npm run build

Деплой на сервер по SSH

      # Деплой через SSH
      - name: Deploy via SSH
        uses: appleboy/ssh-action@v1
        with:
          host: ${{ secrets.SERVER_HOST }}
          username: ${{ secrets.SERVER_USER }}
          key: ${{ secrets.SSH_PRIVATE_KEY }}
          script: |
            cd /var/www/my-app
            git pull origin main
            npm ci --production
            pm2 restart app

Секреты (пароли и ключи)

Пароли и ключи хранятся в Secrets репозитория, не в коде. Settings → Secrets and variables → Actions → New repository secret.

⚠️

Никогда не вставляй пароли прямо в yml-файл. Используй ${{ secrets.MY_SECRET }} — значение подставится при запуске, но не будет видно в логах.

Деплой на GitHub Pages

Для статических сайтов — ещё проще, встроенное действие:

name: Deploy to Pages

on:
  push:
    branches: [ main ]

permissions:
  contents: read
  pages: write
  id-token: write

jobs:
  deploy:
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/configure-pages@v5
      - uses: actions/upload-pages-artifact@v3
        with:
          path: './dist'
      - id: deployment
        uses: actions/deploy-pages@v4
💡

Статус последнего workflow виден прямо на странице репозитория — зелёная галочка или красный крест рядом с коммитом.
