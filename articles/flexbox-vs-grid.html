Flexbox vs Grid: когда что использовать

Вечный вопрос, который путает новичков. На самом деле правило простое — и после этой статьи ты больше не будешь задумываться.

Простое правило

Flexbox — когда элементы выстраиваются в одну линию (строку или колонку).
Grid — когда нужна двумерная сетка (строки И колонки одновременно).

💡

Они не конкуренты — используй оба. Типичная страница: Grid для основного макета, Flexbox для навигации и карточек.

Flexbox — одно направление

Идеален для: навигации, строки кнопок, выравнивания элементов по центру, карточек в ряд.

/* Навигация в ряд */
.nav {
  display: flex;
  align-items: center;
  justify-content: space-between;
  gap: 16px;
}

/* Центрировать элемент по вертикали и горизонтали */
.centered {
  display: flex;
  align-items: center;
  justify-content: center;
}

/* Карточки в ряд, перенос при нехватке места */
.cards {
  display: flex;
  flex-wrap: wrap;
  gap: 16px;
}

Grid — две оси сразу

Идеален для: макета страницы, галерей, сеток с фиксированным количеством колонок.

/* Макет страницы: сайдбар + контент */
.layout {
  display: grid;
  grid-template-columns: 260px 1fr;
  grid-template-rows: auto 1fr auto;
  min-height: 100vh;
}

/* Адаптивная сетка — сами считают сколько влезет */
.grid {
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(280px, 1fr));
  gap: 20px;
}

/* Строгая сетка 3 колонки */
.gallery {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  gap: 12px;
}

Реальный пример: карточки блога

Для сетки карточек — Grid. Внутри каждой карточки (тег + заголовок + дата) — Flexbox.

/* Сетка карточек — Grid */
.posts-grid {
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(260px, 1fr));
  gap: 16px;
}

/* Внутри карточки — Flexbox */
.post-card {
  display: flex;
  flex-direction: column;
  gap: 10px;
}

/* Тег и время чтения в одной строке — Flexbox */
.card-top {
  display: flex;
  align-items: center;
  justify-content: space-between;
}

auto-fill vs auto-fit

Оба создают адаптивные колонки, но по-разному ведут себя когда элементов меньше чем колонок:

  • auto-fill — резервирует пустые колонки (сохраняет размер ячеек)
  • auto-fit — растягивает оставшиеся элементы на всю ширину

Для карточек обычно нужен auto-fill — карточки не растягиваются до огромных размеров.
