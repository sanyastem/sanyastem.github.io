Docker для новичка: контейнеры за 10 минут

«У меня работает» — это не ответ. Docker решает проблему окружения раз и навсегда: приложение работает одинаково на твоём ноутбуке, у коллеги и на сервере.

Что такое контейнер

Контейнер — это изолированный процесс со своей файловой системой, зависимостями и настройками. В отличие от виртуальной машины, он не включает целую ОС — поэтому запускается за секунды и весит мегабайты, а не гигабайты.

Аналогия: контейнер — как zip-архив с приложением и всем необходимым для его запуска.

Установка

Скачай Docker Desktop с официального сайта. Он включает Docker Engine, CLI и GUI. После установки проверь:

docker --version
# Docker version 27.x.x

docker run hello-world
# Если видишь "Hello from Docker!" — всё работает

Основные команды

# Запустить контейнер (скачает образ если нет локально)
docker run nginx

# Запустить в фоне (-d) и пробросить порт (-p)
docker run -d -p 8080:80 nginx
# Теперь открой http://localhost:8080

# Список запущенных контейнеров
docker ps

# Остановить контейнер
docker stop <container_id>

# Список образов
docker images

Dockerfile — свой образ

Dockerfile — это инструкция по сборке образа. Пример для простого Node.js приложения:

# Берём базовый образ
FROM node:20-alpine

# Рабочая директория внутри контейнера
WORKDIR /app

# Копируем package.json и ставим зависимости
COPY package*.json ./
RUN npm install

# Копируем весь код
COPY . .

# Открываем порт
EXPOSE 3000

# Команда запуска
CMD ["node", "index.js"]
# Собрать образ
docker build -t my-app .

# Запустить
docker run -p 3000:3000 my-app

docker-compose — несколько сервисов

Если нужно запустить приложение + базу данных + redis — используй docker-compose.yml:

version: '3'
services:
  app:
    build: .
    ports:
      - "3000:3000"
    depends_on:
      - db
    environment:
      - DATABASE_URL=postgres://user:pass@db:5432/mydb

  db:
    image: postgres:16
    environment:
      POSTGRES_USER: user
      POSTGRES_PASSWORD: pass
      POSTGRES_DB: mydb
    volumes:
      - pgdata:/var/lib/postgresql/data

volumes:
  pgdata:
# Запустить всё
docker-compose up -d

# Остановить
docker-compose down
💡

Добавь .dockerignore рядом с Dockerfile и включи туда node_modules, .git, .env — они не должны попадать в образ.
