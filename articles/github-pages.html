Как запустить сайт на GitHub Pages за 5 минут

GitHub Pages — бесплатный хостинг прямо из репозитория. Никаких серверов, никаких платежей. Просто кладёшь файлы в репо — и сайт уже доступен в интернете. В этом гайде пройдём весь путь от нуля.

Что такое GitHub Pages

GitHub Pages — сервис от GitHub, который берёт файлы из твоего репозитория и раздаёт их как статический сайт. Работает с HTML, CSS, JavaScript. Бесплатно для публичных репозиториев, домен по умолчанию — username.github.io.

Идеально для: портфолио, документации, блогов, лендингов.

Шаг 1 — Создай репозиторий

Для персонального сайта имя репозитория должно быть строго username.github.io, где username — твой логин на GitHub. Именно так, без отступлений.

💡

Для проекта (не персонального сайта) имя может быть любым — он будет доступен по адресу username.github.io/repo-name.

Шаг 2 — Добавь index.html

GitHub Pages ищет точку входа — файл index.html в корне репозитория (или в папке /docs). Вот минимальный файл для старта:

<!-- index.html -->
<!DOCTYPE html>
<html lang="ru">
<head>
  <meta charset="UTF-8">
  <title>Мой сайт</title>
</head>
<body>
  <h1>Привет, мир!</h1>
</body>
</html>

Шаг 3 — Включи Pages в настройках

  • 01
    Открой Settings репозитория

    Вкладка в верхнем меню репозитория.

  • 02
    Найди раздел Pages

    В левом сайдбаре — Pages.

  • 03
    Source: Deploy from branch

    Выбери ветку — обычно main или master, папку — / (root).

  • 04
    Сохрани

    Нажми Save. GitHub начнёт деплой — обычно занимает 1–2 минуты.

Шаг 4 — Открой сайт

После деплоя в том же разделе Pages появится зелёная плашка с адресом: https://username.github.io. Жми и смотри результат.

Кастомный домен

Если хочешь подключить свой домен (mysite.ru), нужно:

  1. В настройках Pages ввести домен в поле Custom domain
  2. У регистратора домена добавить DNS-запись:
    CNAME @ username.github.io
  3. Подождать распространения DNS — от 5 минут до нескольких часов
✅

HTTPS включается автоматически через Let's Encrypt — не нужно ничего настраивать дополнительно.

Частые проблемы

Сайт не обновляется после push

Проверь, что пушишь в ту ветку, которую указал в настройках Pages. Если ветка верная — жди, иногда CDN кэшируется несколько минут.

Страница 404

Убедись, что файл называется именно index.html (строчными буквами) и лежит в корне репозитория, а не в подпапке.

⚠️

GitHub Pages не поддерживает серверную логику (Node.js, PHP, Python). Только статика: HTML, CSS, JS.
