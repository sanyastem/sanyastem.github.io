Терминал: команды которые экономят время

Большинство разработчиков используют 10% возможностей терминала. Вот трюки, которые я использую каждый день и которые реально ускоряют работу.

Ctrl+R — поиск по истории

Нажми Ctrl+R и начни вводить — терминал найдёт последнюю команду с таким текстом. Это быстрее чем листать стрелкой вверх.

# Нажимаешь Ctrl+R, вводишь "docker"
(reverse-i-search)`docker': docker-compose up -d

# Enter — выполнить, стрелки — редактировать

alias — короткие команды

Создай псевдонимы для длинных команд. Добавь в ~/.bashrc или ~/.zshrc:

# Git сокращения
alias gs='git status'
alias ga='git add .'
alias gc='git commit -m'
alias gp='git push'
alias gl='git log --oneline -10'

# Навигация
alias ..='cd ..'
alias ...='cd ../..'
alias ll='ls -la'

# Docker
alias dcu='docker-compose up -d'
alias dcd='docker-compose down'
# Применить изменения в текущей сессии
source ~/.zshrc

&& и || — цепочки команд

&& — выполнить следующую команду только если предыдущая прошла успешно.
|| — выполнить следующую только если предыдущая провалилась.

# Собрать и запустить (только если сборка прошла)
npm run build && npm start

# Установить зависимости и запустить
npm install && npm run dev

# Создать папку и перейти в неё
mkdir my-project && cd my-project

Навигация как профи

# Перейти в предыдущую директорию
cd -

# Перейти домой
cd ~
cd    # то же самое

# История команд
history
history | grep docker   # найти команды с docker

# Выполнить команду из истории по номеру
!42

Полезные горячие клавиши

  • Ctrl+C — прервать текущую команду
  • Ctrl+L — очистить экран (как clear)
  • Ctrl+A — перейти в начало строки
  • Ctrl+E — перейти в конец строки
  • Ctrl+W — удалить слово назад
  • Tab — автодополнение команды или пути
  • Tab Tab — показать все варианты
